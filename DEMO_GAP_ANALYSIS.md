# Demo 当前不足与改进方向

日期：2026-06-19

## 当前结论

第一版已经跑通玩家偏好、问答、剧透确认、来源展示、匿名事件、洞察面板和技术评测页，但整体仍偏可演示原型。下一阶段应优先补齐真实模型调用、证据可信度、外部搜索核验、评测可解释性和公开部署边界。

## 2026-06-19 追加修复

- 修正过期受控知识：移除“桑多涅与阿兰没有直接关系”的旧边界条目，替换为“桑多涅是阿兰晚年造物，形象与记忆源自玛丽安”的双语受控条目。
- 修正检索排序：关系、联系、造物、creation、relationship 等问法会给关系/造物条目额外加权，避免被通用角色公开身份条目压过。
- 新增知识审计测试：禁止重新引入“证据不足以建立桑多涅和阿兰直接关系”的旧结论。
- 接入 DeepSeek tool calling 搜索循环：模型负责提出 `search_web_evidence` 查询，应用执行真实联网搜索、来源分级、工具结果回填和 citation 校验。
- 来源分级改为 `official`、`trusted_wiki`、`community`、`unknown_web`；英文 Fandom、中文 Fandom、BWiki 属于高可信 Wiki，不冒充官方来源。

## 已确认不足

1. 模型调用路径曾不完整（已处理）

   当前实现原本在未配置 `LLM_API_KEY` 时返回 deterministic fallback；外部搜索命中时也会直接返回预设模板，不进入模型生成。这会让现场回答看起来像预设答案，削弱 Agent Demo 的可信度。已改为：配置 DeepSeek OpenAI-compatible Chat API 后，受控证据和外部搜索证据都会进入 chat completions。

2. 外部搜索仍只是联网证据搜索的第一版（已部分处理）

   当前搜索工具真实访问英文 Fandom、中文 Fandom 与 BWiki 的 MediaWiki API，并已从只使用搜索摘要升级为继续抓取页面 intro extract；同时接入轻量通用网页搜索，将贴吧、知乎、NGA、公众号等白名单外来源按 `community` 或 `unknown_web` 低权重处理。生产部署仍建议替换为稳定搜索 API。

3. 证据分层还不够严格（已部分处理）

   规划要求区分“来源身份”和“事实状态”。页面现在同时展示 `sourceKind` 和 fact status，并将 Wiki 来源标为 `trusted_wiki` / 高可信 Wiki，避免把社区索引页面展示成官方站点。后续仍应补充官方页面或游戏内原文出处。

4. 检索仍依赖手写别名和 substring 匹配

   当前受控检索依靠实体别名、关键词展开和简单分数排序。它能覆盖脚本内问题，但对现场自由问法、同义表达和复杂跨语言问题不够稳。规划中的 PostgreSQL 全文检索、查询改写和更完整的低置信度回退尚未落地。

5. 剧透门控存在表达绕过风险（已处理）

   高风险剧透确认已从“问题关键词 + 检索到 Level 3 条目”改为独立 spoiler intent 优先拦截；即使没有检索到 Level 3 证据，结局或身份反转类请求也会二次确认。

6. 生成结果缺少结构化校验（已处理第一层）

   模型现在要求返回 `{ answer, citedSourceIds }` JSON，并校验 cited source id 必须来自本次证据，否则降级到 deterministic fallback。后续仍可继续做更细的 claim-level 逐句校验。

7. 洞察建议仍是模板化草稿

   当前洞察信号由确定性规则触发，但推荐文案是固定模板，并非规则触发后由模型结合聚合证据起草。它可以演示闭环，但还不能充分体现“证据到行动”的定制化价值。

8. 技术评测页缺少人工审核材料（已处理）

   评测结果现在返回完整回答、citation 摘要和失败检查项；页面可展开查看回答与引用，支持人工审核。

9. 公开部署边界仍需补齐（已部分处理）

   Chat、剧透确认和评测 API 已加入进程内轻量限流。本地 JSON 事件存储仍只适合单实例 Demo；Supabase 配置、线上故障降级、密钥轮换和平台级限流仍建议在公开评审版本补齐。

## 建议优先级

### P0

- 保证所有非 fallback 回答真实调用 DeepSeek OpenAI-compatible Chat API。
- 修正证据条目的来源和事实状态标注。
- 评测页展示回答、引用、失败原因和人工审核入口。

### P1

- 增强外部搜索：抓取页面正文、引用和官方出处。
- 独立实现 spoiler intent 分类，减少关键词绕过。
- 增加中英文自由问法回归测试。

### P2

- 将洞察建议从固定模板升级为“规则触发 + 证据约束模型起草”。
- 接入 Supabase 作为公开部署默认事件存储。
- 增加 API 限流、超时和用户可见降级提示。
