# 至冬关系图谱：左右阵营冲突布局设计

## 目标

将现有至冬关系图从容易产生文字、节点和连线遮挡的自由星图，改为清晰、稳定且具有叙事张力的左右阵营图。

新布局需要同时满足：

- 一眼看出冰之女皇统领愚人众及十一席执行官。
- 一眼看出冰之女皇与天理的对抗。
- 清楚呈现神之心、第三降临者、严冬计划、坎瑞亚与深渊等关键线索。
- 所有节点、关系标签和装饰背景互不遮挡。
- 桌面端完整、规整；窄屏可横向浏览且不会造成页面级横向溢出。
- 保留当前节点点击、右侧详情、关系分析、状态颜色和头像功能。

## 视觉方向

整体采用“冰封战略档案”的视觉语言，而不是普通企业组织图。

画面由两块彼此对峙的阵营区域组成：

- 左侧为至冬与愚人众，使用偏冷的灰蓝、冰青和深海军蓝。
- 右侧为天理与世界秩序，使用偏暖的象牙白、淡金和低饱和灰。
- 中央为一条狭窄的“冲突裂隙”，承载阵营对抗和神之心争夺关系。

背景只提供分区和氛围，不承担信息表达。背景纹理、圆环和装饰线的对比度必须显著低于节点与文字。

## 信息架构

### 左侧：至冬阵营

左侧按从上到下的层级组织：

1. 冰之女皇
2. 愚人众
3. 统括官丑角
4. 十一席执行官

严冬计划作为冰之女皇的战略分支，放在女皇上方或左上方的独立小节点中，通过短线连接，不占用主组织树通道。

十一席采用统一宽高的双排阵列：

- 第一排：第一席至第六席
- 第二排：第七席至第十一席

第十席保留占位节点。特殊命运角色继续使用状态底色区分，但所有执行官节点保持相同尺寸、字体基线和席位编号位置。

### 中央：冲突轴

中央区域只放置跨阵营的核心关系，不放普通角色节点。

核心元素包括：

- 冰之女皇与天理的双向对抗线。
- 神之心焦点节点。
- “收集神之心”关系标签。

冲突轴需要保持足够宽的留白。任何标签都不能直接压在线条或节点边缘上。

“收集神之心”放在冰之女皇通向神之心的独立线路上，使用带实体背景的关系胶囊，而不是裸露 SVG 文本。

### 右侧：天理与世界秩序

右侧按叙事关系组织：

1. 天理位于右上方，和冰之女皇保持同一视觉高度。
2. 神之心位于中央偏右，作为冲突焦点。
3. 第三降临者位于神之心下方，通过垂直关系连接。
4. 坎瑞亚与深渊位于右下方，作为旧灾变和世界外力量背景。

右侧不构建复杂组织树，只呈现三到四条最关键的世界观关系，避免信息密度与左侧十一席竞争。

## 布局与尺寸

桌面画布使用固定设计坐标，推荐尺寸约为 `1180 × 760`，并根据容器宽度缩放或在较窄容器中内部滚动。

画布划分为：

- 左阵营区：约 52%
- 中央冲突轴：约 16%
- 右阵营区：约 32%

节点之间必须保留最小安全距离：

- 横向最小间距：24px
- 纵向最小间距：32px
- 关系标签与任一节点边缘：至少 14px
- 画布边缘与节点：至少 24px

十一席节点应采用较紧凑的尺寸，避免左侧过度膨胀。组织层级节点可以略宽于角色节点，但不能使用夸张的大字号造成头重脚轻。

## 节点设计

所有节点必须拥有不透明或接近不透明的底色，避免背景线和关系线穿透文字。

节点视觉层级：

- 冰之女皇和天理：阵营主节点，宽度最大，使用轻微辉光和阵营色顶边。
- 愚人众和丑角：组织层节点，使用更克制的矩形样式。
- 执行官：统一卡片样式，左上角显示席位编号，中央显示称号，底部显示当前状态。
- 世界观节点：较轻的文献索引样式，避免与角色节点混淆。

节点状态颜色沿用现有语义：

- 现役：冷绿色
- 前席或脱离：冰蓝
- 死亡或身体毁坏：暗红
- 休眠：低饱和紫
- 未知：灰色虚线

颜色只用于底边、状态点或浅色背景，不使用高饱和整块填充。

## 连线系统

连线分为三类：

1. 统领线：冰之女皇 → 愚人众 → 丑角 → 十一席
2. 冲突线：冰之女皇 ↔ 天理
3. 世界观线：冰之女皇 → 神之心 → 第三降临者，以及丑角 → 坎瑞亚与深渊

连线采用正交折线或柔和的短曲线，不使用长距离穿过节点群的对角线。

所有连线位于节点下层。跨区域连线必须经过预留通道，不得穿过任何节点。

装饰线和真实关系线应明显区分：

- 真实关系线：清晰、对比度较高、有箭头。
- 装饰线：低透明度、无箭头、不可与节点相交。

## 关系标签

关系标签不再作为裸 SVG 文本绘制。

每个显示标签的关系使用独立 HTML 或 SVG 背景组：

- 浅色不透明背景
- 细边框
- 小号粗体文字
- 4–8px 内边距
- 轻微阴影或描边

标签位置由关系数据显式配置，可设置横向和纵向偏移。不能只依赖两节点中点自动定位。

需要显示的标签保持精简：

- 对抗
- 统领
- 推动严冬计划
- 收集神之心
- 遗骸铸成
- 坎瑞亚旧事

其他显而易见的组织树关系可只保留箭头，不重复写文字。

## 图例与说明

状态图例移出地图画布，放在地图标题下方，与操作提示并列。

图例不再覆盖左下角节点或关系线。窄屏拖动提示也放在标题区域，不占画布内部空间。

地图内部仅保留必要的阵营标题：

- 至冬 / 愚人众
- 冲突轴
- 天理 / 世界秩序

阵营标题使用低对比度大字作为分区水印，不参与交互。

## 交互

保留当前交互逻辑：

- 点击节点更新右侧详情。
- `Shift + 点击` 选择两个节点分析关系。
- 特殊角色显示席位和命运状态。
- 详情弹窗、文本线索和推荐追问继续使用。

新增或调整：

- 节点悬停时，仅高亮与该节点直接相连的关系线，其他线降低透明度。
- 键盘聚焦状态与悬停状态一致。
- 选中节点使用清晰外框，不改变节点尺寸，避免布局抖动。

## 响应式策略

### 桌面端

- 画布完整展示左右阵营。
- 右侧详情栏保持当前结构。
- 图谱区域内部不得出现文字裁切或节点重叠。

### 平板端

- 图谱和详情改为上下排列。
- 图谱保持固定最小宽度，在自身容器内横向滚动。
- 初始滚动位置尽量对准冰之女皇、冲突轴与天理，而不是最左边缘。

### 手机端

- 页面本身不产生横向滚动。
- 地图视口内部可左右拖动。
- 节点文字不缩小到难以阅读；宁可保持尺寸并允许内部滚动。
- 图例、拖动提示和阵营说明位于视口外部。

## 防遮挡机制

实现时必须同时采用静态设计约束和自动检查。

### 静态约束

- 每个节点使用明确的设计坐标。
- 每个可见关系标签使用明确的设计坐标或偏移。
- 节点、标签和图例分别使用独立图层。
- 禁止背景装饰使用高于关系线的层级。

### 自动检查

增加布局测试或纯函数检查：

- 所有节点坐标位于画布范围内。
- 执行官席位不重复。
- 节点矩形之间不存在重叠。
- 可见关系标签的矩形不与任一节点矩形重叠。
- 图例不在地图画布内部。
- 核心边数量保持精简。

自动检查只验证几何安全线，最终仍需要桌面和手机截图检查。

## 验收标准

设计完成后的页面应满足：

- “收集神之心”等关系文字完全可见。
- 任意关系标签都不被节点、详情框或其他文本覆盖。
- 任意节点文字都不被连线或背景装饰穿过。
- 左侧愚人众与右侧天理形成明确对峙，整体视觉重心平衡。
- 十一席结构规整且可以快速扫描。
- 特殊命运角色仍然容易识别。
- 桌面端和手机端均无页面级横向溢出。
- 点击、关系分析和详情功能无回归。

## 非目标

本次不修改：

- 节点事实内容和人物状态结论。
- 角色头像资源。
- 关系分析后端逻辑。
- 整站配色和其他页面布局。
- 新增更多关系边或角色节点。
